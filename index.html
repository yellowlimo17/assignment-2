    • Node::appendChild()

Debugging

This is perhaps the best time to start using the builtin browser debugger explained in class at the beginning of the semester.

  1. Open the browser DevTools
  2. Switch to the "Source" tab, the left-side panel should show the list of HTML, CSS, and JS files loaded by your browser.
  3. Select jsdom.js to start debugging your JavaScript program. The following screenshot shows Chrome DevTools after a breakpoint is set at line 16 of your program and the page is reloaded. Breakpoints can be added/removed by clicking the line numbers.

There are six important components in the debugger panel

  • Step Over: Continue to the next line after the current statement
  • Step Into Function: if the next line is a function call, trace the execution inside that function
  • Step Out: Resume the current function and return to its caller
  • The leftmost button (to the left of "Step Over") has dual purposes: Pause or Resume debugging
  • The "Scope" tab where you can inspect all the variables used by your program
  • The "Console" tab (lower right) where all the console.log() output will be displayed.
Be sure you are aware that console.log() does not send its output to the browser "canvas", and it is mainly used for "debugging" purposes during development. On a real production release, your JS code should be free of any console.xxx() statements.

(7 pts) Part 3: Array of objects

In this part, your JS code will be handling an array of objects. The concept is pretty much similar to Java arrays (not Java ArrayList). Let's say you have an array of two items: 

    const months = [
      {name: "January", days: 31},    
      {name: "February", days: 28}
    ];

    console.log(months[0].name); // Output: January
    console.log(months[0].days); // Output: 31

Given an array of objects atomObjects,:

  • Create a numbered list of atom name and its weight. The completed HTML should look approximately like the following (except for randomized order)
    1. Aluminum (weight: 26.982)
    2. Barium (weight: 137.33)
    3. ...many more...
  • Define two classes in your CSS: "light" and "heavy". Apply the "heavy" class to atoms whose weight is above 150. Apply different text colors to each class:
    • Use the HSL color space to define the colors
    • The two colors should share the same Hue and Saturation
    • The "light" class should use a higher Luminance and the "heavy" class should use a lower luminance. Be sure to use non-zero luminance, otherwise the color turns black.
  • Extra Credits: Explore how to print the weight to show only one decimal place so the output will look like
    1. Aluminum (weight: 27.0)
    2. Barium (weight: 137.3)
    3. ...many more...
